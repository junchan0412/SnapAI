# SnapAI 1.6.8

SnapAI 1.6.8 聚焦两个高频体验问题:默认动作快捷键不完整,以及部分应用中选中文字后仍提示“未检测到选中文本”或替换结果被追加到原文后。此版本补齐默认快捷键体系,并重构跨应用取词与文本写回策略。

## 新增

- 默认动作新增完整快捷键:
  - 提问: `Option + A`
  - 翻译: `Option + T`
  - 润色: `Option + P`
  - 总结: `Option + S`
  - 解释代码: `Option + E`
  - 快捷提问面板: `Option + Space`
- 设置页“动作”区域新增“恢复默认快捷键”。
- 恢复默认快捷键会重置默认动作和快捷提问面板,并清理占用这些默认组合的自定义动作快捷键。

## 改进

- Accessibility 取词现在会遍历更深的目标应用可访问性树。
- 新增对 `AXStringForRange` 参数化读取的支持,提升浏览器、富文本编辑器和复杂控件中的选区读取成功率。
- 剪贴板复制兜底最多重试 3 次,降低目标应用响应慢时的误判。
- 复制兜底现在复用更完整的 pasteboard 文本解析能力,支持 UTF-8/UTF-16、RTF、HTML 和部分旧式文本类型。
- 文本替换不再按原文字符数模拟 `Shift + Left` 重选,避免选区丢失时把结果追加到错误位置。
- 剪贴板捕获和 macOS Services 捕获会信任目标应用保留的原选区,直接进入安全粘贴路径。

## 测试

- 补齐默认动作快捷键、恢复默认快捷键和迁移补齐快捷键测试。
- 补齐替换准备策略测试,固定“不再合成方向键重选”的行为。
- 补齐取词重试与 AX 遍历预算测试。
