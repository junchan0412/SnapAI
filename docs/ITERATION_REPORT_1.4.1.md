# SnapAI 1.4.1 UI/UX 迭代报告

## 目标

1.4.1 聚焦现有 UI/UX 的最终打磨,不改变 1.4.0 已完成的本地模型、隐私 fallback、写回兼容和诊断能力。核心目标是:

- 减少高频窗口的视觉厚重感。
- 让窄宽度下的工具条和状态区不再挤压变形。
- 强化结果面板的追问与路由状态可见性。
- 统一设置页、历史窗口和健康中心的紧凑桌面风格。

## 已完成

### 1. 统一轻量样式层

- 新增 `SnapAIUI.swift`,提供紧凑卡片、状态胶囊和图标按钮样式。
- 统一多个窗口中的圆角、描边、填充透明度和小按钮尺寸。
- 继续使用系统语义颜色和 material,保持浅色/深色模式自适应。

### 2. 设置页密度优化

- 设置窗口从 660x520 收敛到 640x500。
- 弱化外层内容容器,减少“卡片套卡片”的厚重观感。
- 供应商、动作、历史统计、工作模式和通用设置分区统一为更轻的紧凑 surface。
- 当前使用中的供应商改为状态胶囊,信息层级更明确。

### 3. 结果面板交互优化

- 追问输入从底部操作按钮中拆出,并新增显式发送按钮。
- 结果操作按钮收束为稳定工具条,降低窄窗口下的拥挤感。
- 顶部新增模型与路由状态条,生成中、当前模型和 fallback 状态更容易被看见。

### 4. 历史窗口扫读优化

- 搜索、收藏、重置和导出/上下文操作保留在第一行。
- 动作、模型和标签筛选移动到第二行,避免单行拥挤。
- 当前筛选摘要改为状态胶囊,用户能更快理解当前列表为什么被筛选。
- 历史记录卡片更轻,操作图标尺寸更稳定。

### 5. 权限健康中心稳定性优化

- 健康状态区从 Grid 行改为自适应卡片,避免文字和按钮在窄宽度下变形。
- 最近请求和签名信息支持多行展示。
- 复制精简、复制完整和重新检测入口保留在顶部。
- 详情区域使用统一轻量 surface,提高诊断信息的可读性。
