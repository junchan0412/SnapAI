# SnapAI 1.5.0 迭代报告

## 目标

1.5.0 的目标是完成 SnapAI 从“可配置 AI 菜单栏工具”到“可沉淀工作流的系统级文本助手”的第一段升级。核心方向包括上下文包/项目记忆、历史知识库、命令面板主入口和可分享的模板市场/动作库。

## 已完成

### 1. 项目记忆

- 上下文包可保存项目背景、术语表、写作风格或代码栈偏好。
- 当前上下文会合并进实际 System Prompt,并在结果面板和诊断中展示可读状态。
- 支持通过 URL Scheme 和命令面板切换、复制、清空或查看上下文状态。
- 可从筛选后的历史记录创建或更新上下文包,让历史沉淀为项目记忆。

### 2. 历史知识库

- 历史窗口支持搜索、收藏、删除、标签编辑和按动作/模型/标签筛选。
- 支持导出全部或筛选后的历史 Markdown。
- 支持从收藏、标签或搜索结果创建上下文包。
- 隐私相关历史标签会保留在筛选和导出中,便于审计哪些内容经过脱敏、预览或仅保存元信息。

### 3. 命令面板主入口

- `Command + K` 命令面板可搜索动作、模型、上下文包、历史、工作模式、设置项、显示行为、诊断和更新入口。
- 结果面板行为、历史导出、历史转上下文、权限诊断和路由偏好都已有命令面板入口。
- 1.5.0 新增动作模板安装入口,用户可以直接搜索模板名称或“动作库”完成安装。

### 4. 模板市场 / 动作库

- 新增 `ActionTemplateLibrary`,统一维护内置动作模板、分享包格式、导入清洗和安装去重。
- 内置模板包括邮件回复、会议纪要、代码审查、中英双语润色和图片理解。
- 动作设置页新增导入/导出动作库,JSON 分享包可用于备份、迁移和社区分享。
- 分享动作库会自动清除外部快捷键、供应商绑定和模型覆盖,避免导入后污染用户本机配置。
- 导入旧版纯 `[AIAction]` JSON 仍可兼容,便于从早期配置迁移。

### 5. 测试覆盖

- 新增动作模板库测试,覆盖内置模板完整性、分享包便携性、导入清洗和重复名称/id 处理。
- 保留既有版本比较、Release tag 解析、更新校验、热键冲突、Base URL 规范化、模型 fallback、Prompt 渲染、隐私脱敏、历史导出和上下文构建测试。

## 后续方向

- 1.6.0 将重点优化“任意应用选中文字后, SnapAI 能理解上下文并执行动作”的完整链路。
- 下一阶段会对文本捕获、写回、剪贴板保护、前台应用恢复、失败诊断和代码结构进行全量审查。
- 文档会进一步扩展为完整使用手册,并使用最新版 UI 图片更新截图。
