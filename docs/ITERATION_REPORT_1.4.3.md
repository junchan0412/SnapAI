# SnapAI 1.4.3 迭代报告

## 目标

1.4.3 聚焦两个用户可见问题:

- 设置窗口右上角置顶图标点击后没有即时反馈,用户无法判断状态是否已经变化。
- AI 路由卡片只占左侧内容宽度,右侧留下大面积空白,破坏设置页的视觉平衡。

## 已完成

### 1. 置顶状态即时反馈

- 引入 `SettingsWindowPinState` 作为设置窗口置顶状态的 SwiftUI 可观察源。
- 设置页按钮点击时先更新 `pinState.isPinned`,再通知 AppDelegate 应用窗口层级。
- AppDelegate 在任何入口修改置顶状态时也会回写 `pinState`,保证按钮、菜单和命令面板状态一致。
- 保留状态图标语义:未置顶为空心图钉,已置顶为实心图钉。

### 2. AI 顶部区域重构

- 用 `AI 配置` 概览卡片替代原先独立的“当前使用”和“AI 路由”两张卡片。
- 左侧展示当前供应商、模型和端点摘要。
- 右侧展示自动路由开关、路由偏好和 fallback 开关。
- 顶部状态胶囊展示自动路由 / 固定模型与 fallback 状态。
- 双列布局让右侧空间承载真实信息,消除大块留白。

### 3. 设置页一致性打磨

- 顶部概览、供应商卡片和动作卡片增加明确的横向展开约束。
- 控件密度保持桌面设置页风格,减少大块空白与宽度跳动。
- 继续使用系统语义颜色和轻量 surface,保持浅色/深色模式自适应。
