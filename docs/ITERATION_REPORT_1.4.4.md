# SnapAI 1.4.4 迭代报告

## 目标

1.4.4 作为进入 1.5.0 大功能前的优化版本,核心目标是提升 AI 路由、上下文和结果面板的可理解性。自动路由已经具备较强诊断能力,但用户日常界面只能看到很短的 route note,无法判断“为什么选择这个模型”。

## 已完成

### 1. 路由透明度

- 新增 `AIRequestDiagnostics.visibleRouteExplanation`,把后台诊断转化为适合 UI 展示的中文摘要。
- 摘要包含实际优先请求路由、路由原因、自动路由偏好、fallback 状态、上下文大小、token 估算、图片输入和预检跳过数量。
- 新增 `AIRequestDiagnostics.visibleRouteStatusTitle`,让 UI 可以稳定显示固定模型、自动路由、自动路由 + Fallback 或无可用模型。

### 2. 结果面板优化

- 结果面板顶部路由栏不再只显示模型和简短 note。
- 当前上下文包会以状态胶囊展示,用户能确认请求是否合并了项目记忆。
- 路由解释支持两行显示,在不显著增加面板高度的前提下提供更多判断信息。

### 3. 设置页路由预览

- AI 模型设置页新增实时路由预览。
- 预览会根据当前动作、自动路由、fallback、当前上下文和可用模型计算优先路由。
- 当没有可用路由时,预览会直接提示检查供应商、API Key 和模型启用状态。

### 4. 测试覆盖

- 新增路由解释测试,覆盖自动路由、fallback、上下文、图片输入、token 估算和预检跳过。
- 保留既有路由诊断、预检跳过、错误恢复和构建检查。
